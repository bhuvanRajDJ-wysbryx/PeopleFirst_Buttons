Button Design System

Defines how PeopleFirst presents actions, navigation, and feedback across the platform. This reference establishes visual hierarchy, interaction behavior, and labeling standards so every module delivers a consistent, confident experience.

Foundation components

Button

Use: Every actionable control in forms, dialogs, tables, and page headers.
Appearance: Full-round pills for actions; subtle lift and deepened color on hover for primary actions; muted fill on secondary and outline styles.
Rationale: A single button language keeps hierarchy readable and reinforces trust in high-stakes workflows.

Primary action patterns

Primary action with icon

Use: The main task on a page when the action is widely recognized (add, save, approve, download).
Appearance: Brand-blue filled pill with a leading icon and verb-first label.
Rationale: Accelerates recognition and signals commitment for frequent, high-value actions.

Compact toolbar control

Use: Row actions and toolbars where space is limited and the meaning is established (edit, delete, export).
Appearance: Circular outline control with tooltip on hover and focus—never rely on the browser title attribute alone.
Rationale: Preserves table density while keeping actions discoverable and accessible.

Primary action, text only

Use: High-priority actions whose meaning is specific to context and has no suitable universal icon.
Appearance: Same brand-blue pill as the icon variant, label only.
Rationale: Maintains primary emphasis without forcing decorative or ambiguous icons.

Secondary action

Use: Cancel, back, clear filters, and non-destructive alternatives that support the primary path.
Appearance: Light outlined pill; no icon required.
Rationale: Reduces visual competition so the primary action remains the clear focal point.

Reversible removal uses a secondary Delete. Permanent removal uses destructive styling only inside a confirmation dialog.

Supporting interaction patterns

View selector

Use: Switch among related views, filters, or roles where exactly one option is active.
Appearance: Segmented control with orange active segment, or a labeled toggle between Employee and Approver contexts.
Rationale: Communicates mode change instantly without the weight of full navigation tabs.

Primary action with menu

Use: A single default outcome with additional related options (save, submit, close).
Appearance: Primary pill with integrated menu affordance on the trailing edge.
Rationale: Keeps the common path one click away while exposing variants without cluttering the header.

Menu lists alternate outcomes: Save as draft, Save and submit, Save and close.

Bulk selection bar

Use: One or more rows selected in a list or table.
Appearance: Persistent bar with selection count and grouped approve/reject actions.
Rationale: Makes multi-record operations explicit and eliminates repeated row-level clicks.

Inline text action

Use: Supplementary actions embedded in body copy (policy links, see all, learn more).
Appearance: Text styled as a link; underline on hover and focus.
Rationale: Supports the narrative without competing with primary page actions.

Navigation patterns

Module navigation tabs

Use: Top-level sections within a module (overview, staffing, timesheets).
Appearance: Underline style with orange active indicator, or filled orange segment on a white track; optional count pill for workload.
Rationale: Separates major areas without competing with brand-blue action buttons.

Section filter tabs

Use: Filters within a module—manager queues, staffing status, record sections.
Appearance: Inactive tabs on muted ground; active tab on white with orange label, subtle shadow, and a fine bottom indicator; optional count on the right.
Rationale: Surfaces workload at a glance and anchors list content to the selected filter.

Interaction principles

Action placement

Place secondary actions (Cancel, Reject) on the left and primary actions (Save, Approve) on the right in headers and footers. In confirmation dialogs, place Cancel first, then the destructive action.

Access control

Hide controls the user cannot use. When access is blocked but visible, disable the control and explain why in a tooltip. Never disable without a stated reason.

Completion feedback

Confirm success with a toast and preserve page context. You may briefly change the label to reflect completion, then restore the default. Clear loading indicators promptly.

Failure and retry

On failure, show an error message and re-enable the control. Offer Try again as a secondary action when automatic retry is not in effect.

Loading

Show a spinner with an active verb (Saving, Approving, Deleting). Disable all actions in the same group while work is in progress. Use one loading indicator per action group.

Icon placement

Place icons before the label for standard actions. Reserve trailing icons for external links and outbound navigation.

Deletion hierarchy

Style reversible removal as a secondary action. Reserve destructive fill for permanent deletion inside a dedicated confirmation step. Toolbar delete icons remain neutral, never destructive red.

Confirmation depth

Confirm low-risk row removal inline. Require a modal for entity-level or irreversible deletion.

Responsive layout

Stack primary actions full width on narrow viewports in forms and dialogs. Keep toolbar controls compact; allow bulk action bars to wrap.

Navigation vs selection

Reserve orange tabs for module and section navigation. Use segmented controls and role toggles for filters, time ranges, and view modes. Apply one pattern per hierarchy level.

Selection state

Allow only one active item per tab or segment group. Mark the active item with consistent color, weight, and—where applicable—a count badge.

Keyboard and focus

Expose a visible focus ring on every interactive control. Display tooltips when icon actions receive keyboard focus. Provide an accessible name for every icon-only action.

Localization

Mirror directional icons in right-to-left layouts. Keep count badges after the label in all locales.

Workload indicators

Apply count badges only to signal pending work. Omit the badge when the count is zero.

Interaction states

Standard feedback for in-progress, blocked, successful, and destructive actions.
